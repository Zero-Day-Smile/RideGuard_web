# RideGuard Dashboard

## Overview

This is a Next.js dashboard for RideGuard's weekly income protection workflow.
It is focused on two operating modes:
- Rider View
- Admin View

## Implemented Features

1. Weekly premium studio
- Coverage plan selector (Basic, Standard, Plus)
- Dynamic premium calculator driven by:
	- Location risk weight
	- Weather probability factor
	- Income variability score
	- Reliability discount

2. Parametric trigger watch
- Real-time style signal cards for weather, platform uptime, traffic, and delivery activity
- Trigger timeline showing disruption events and claim state progression

3. Rider analytics
- Weekly protected earnings snapshot
- Latest auto-payout status
- Policy status card

4. Admin risk operations
- Fraud alert queue with severity tags
- City-level risk heat bands

5. Product-aligned visuals
- Mobile-responsive layout
- Staggered load animations
- Design language aligned with RideGuard concept (safety, urgency, trust)

## Tech Stack

- Next.js (App Router)
- React + TypeScript
- CSS (custom design system, no UI framework dependency)

## Run Locally

```bash
npm install
npm run dev
```

Open:
- http://localhost:3000

## File Map

- `app/page.tsx`: main rider/admin dashboard UI and interactions
- `app/data.ts`: mock analytics, trigger, and fraud datasets
- `app/globals.css`: visual system and responsive styling
- `app/layout.tsx`: metadata and font wiring

## Mock API Routes

- `GET /api/rider/summary`: rider snapshot plus weekly metrics
- `GET /api/admin/overview`: admin metrics, fraud alerts, city risk bands
- `GET /api/triggers`: disruption signals and trigger timeline
- `POST /api/pricing/quote`: dynamic weekly premium and risk score
