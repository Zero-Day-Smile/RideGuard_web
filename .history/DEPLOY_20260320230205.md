# Deploy RideGuard (Frontend + Backend)

This guide deploys both services from GitHub using Render.

## 1. Push to GitHub

1. Create a GitHub repository.
2. Push this project to the repository.

## 2. Deploy with Render Blueprint

1. Sign in to Render.
2. Click `New` -> `Blueprint`.
3. Connect your GitHub repository.
4. Render will detect `render.yaml` and create:
   - `rideguard-api` (FastAPI backend)
   - `rideguard-web` (Vite static frontend)
5. Click `Apply` to deploy.

## 3. Update URLs after first deploy

If your Render service URLs differ from defaults, update two env vars in Render:

1. Backend service `rideguard-api`:
   - `CORS_ORIGINS` = your frontend URL
   - Example: `https://rideguard-web.onrender.com`

2. Static service `rideguard-web`:
   - `VITE_API_BASE_URL` = your backend URL
   - Example: `https://rideguard-api.onrender.com`

Then trigger a redeploy of the frontend service.

## 4. Verify

1. Open frontend URL and confirm pages load.
2. Open backend health URL:
   - `https://<your-backend-domain>/health`
3. Confirm API data on dashboard page.

## Notes

- Current backend uses SQLite by default (`DATABASE_URL=sqlite:///./rideguard.db`).
- For persistent production data, switch to managed Postgres and set `DATABASE_URL`.
